Cuando: Integration and Querying of Data Over Time

Cuando is system that integrates information over time and resolves conflicts using user-defined preferences. We used it to build historical profiles of security ownership by corporate executives and board members using public SEC data.

Command-line Interface

We have implemented a command-line interface (CLI) that takes user input and passes it to the Cuando API. This interface is used to perform the experiments discussed in Section. The interface offers three commands: init accepts an archive name and three JSON documents describing the schema, resolution dimensions, and preference rules, respectively. The merge command accepts an archive name and a single JSON document describing the information to be integrated into the archive. Finally,the query command accepts an archive name and a single JSON document describing the query.

Queries

Cuando queries are made up of an archive name and three clauses: project, select, and timeslice. Cuando uses the archive name to retrieve the schema from the archive’s metadata, and to direct MongoDB to query the correct set of documents.

The project clause is used to prune parts of documents from the result. It consists of a set of paths:

[path $P_1$, path $P_2$, ..., path $P_n$]

The documents in the result are pruned so that only the paths (and their descendants) in this clause appear in the result. By default, the set of paths is populated by all paths in the schema, which means documents are not pruned. For instance, our SEC documents have two subtrees, one for transactions (/securities/) and another for corporate relationships (corporateRelationships). When we want to see only transactions in the query result, our project clause becomes: [ /securities/ ].

The select clause is used to include only matching documents in the result. It consists of a set of tuples:

[(path $P_1$, value $V_1$), ..., (path $P_n$, value $V_n$)]

Each path must have the SetOf type. Only those Records whose key matches the value is returned. The following select clause will find Freddy Gold’s document in our SEC archive: [ (/reportingOwner/rptOwnerName/, Freddy Gold) ].

The timeslice clause is used to limit query results to information considered true within the temporal interval on a particular temporal dimension. It consists of a single tuple:

(dimension $D$, start time $T_s$, end time $T_e$)

Because temporal intervals are closed-open by convention, the start time must be strictly less than the end time. For example, to return only information during the 2012 US fiscal year, which runs from October 1, 2011 to September 30, 2012, we specify (2011-11-01, 2012-11-01)
